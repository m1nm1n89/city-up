## 入力設計(確定)

### チェックボックス(毎日)
- 3個固定(10候補からユーザーが選ぶ)
- 「次に挑戦したいリスト」を別途設置(メモ枠)

### 進捗の上限設計
- 最終目標 → 具体タスク → 1日上限の3層で設定
- 上限超え分は完全に無視(報酬なし、カウントなし)
- テンプレA / テンプレB / 自由書き込みの3択で設定

### 目標入力フロー
- **初回のみ**: 最終目標を入力(例: 月5万円の収益を出す)。テンプレ or 自由記述
- **毎週**: 今週の小さな目標を入力(例: noteを1回投稿する、7回ポストする)。テンプレ or 自由記述
- **週末**: 進捗を記入。達成判定 → コイン獲得 / 未達成 → 何も起きない(罰なし)

### 数値入力(月1回)
- 必須2項目: 今月の投稿数 / 今月の収益(0円可)
- 任意項目: フォロワー数 / PV数 / その他(自由記述)
- 催促あり(未入力では次の画面に進めない)
- 前月値プリセット、「先月と同じ」ボタンで入力体験を軽くする

## マイルストーン設計(確定)

| タイミング | 内容 |
|---|---|
| Day 1 | 建物1棟目が建つ(オンボーディングの派手バージョン) |
| Day 7 | 住人1人目登場 + メンターから「1週間おめでとう」 |
| Day 30 | 古代に進化 + 過去比較機能解放 |
| Day 90 | 中世に進化 + 月次サマリー自動生成解放 |
| Day 180 | 近代に進化 + シェアカード解放 |
| Day 270 | 現代に進化 |
| Day 365 | 未来に進化(卒業演出)。以降は日数以外の変化なし |

### 判定基準
- マイルストーンは **作業日数(累計)** だけで進む。成果入力は不要
- Day 7 は「7回チェックした時点」で発火(連続でなくOK)
- Day 365以降はアプリは使い続けられるが、日数以外の変化はない設計(卒業を組み込む)

## コインのレート設計(確定)

### 入手
| アクション | 獲得コイン |
|---|---|
| チェック1個 | 1コイン |
| チェック3個全達成 | +1ボーナス = 計4コイン |
| 週次目標達成 | 3コイン |
| 月次成果入力 | 5コイン |
| 1日の上限 | 4コイン |

月フル達成で約137コイン/月。

### 使用
| 使い道 | 必要コイン |
|---|---|
| 装飾アイテム(小) | 10コイン |
| 装飾アイテム(大) | 30コイン |
| マイルストーン加速(+1日分) | 10コイン |

### 加速の上限
- 各マイルストーンの最大20%前倒しまで

| マイルストーン | 通常 | 最大加速 | 最短到達 |
|---|---|---|---|
| 古代(Day 30) | 30日 | 6日分 | Day 24 |
| 中世(Day 90) | 90日 | 18日分 | Day 72 |
| 近代(Day 180) | 180日 | 36日分 | Day 144 |
| 現代(Day 270) | 270日 | 54日分 | Day 216 |
| 未来(Day 365) | 365日 | 73日分 | Day 292 |

### 設計思想
- コインの使い道は装飾か加速か、ユーザーが自由に選ぶ(ゲーム性のための競合)
- ガチャはMVPには入れず、リリース後に判断

## UI演出の詳細(確定)

### 時代軸(長期)

原始 → 古代(弥生・竪穴式住居) → 中世(産業革命・煙突街) → 近代(レンガ造り) → 現代(ガラスビル) → 未来(空飛ぶ車・UFO・カラフル)

- 建物が増えるほど住人も増える(各時代の衣装で歩く)
- 住人は横歩き、建物から出入り、荷物の増減あり

### 季節軸(短期・ボタンで切り替え)
- 春: 桜
- 夏: 回転する太陽
- 秋: 焼き芋の煙
- 冬: 雪・雪だるま

### アニメーション階層化
- **普段**: 控えめなフェードイン(0.8秒)
- **マイルストーン達成**: フラッシュ+時代進化(2秒、派手)
- **ボイーン感**: scale 0→1.5→0.7→1.15→0.95→1 + キラキラ

## シェアカード設計(確定)

**位置づけ**: 月次サマリー機能の一部。バイラルループの動線。

### 表示する情報
| 項目 | 内容 |
|---|---|
| 期間 | 「Month 3」「Day 90達成」など |
| 街の現在の姿 | アプリ画面の街並みをそのまま切り出し |
| 累計チェック数 | アプリ内の数字 |
| 建物数 | アプリ内の数字 |
| 住人数 | アプリ内の数字 |
| アプリ名・URL | 小さく(バイラル動線) |

### サイズ
- X用: 16:9 横長
- Instagram/note用: 1:1 正方形
- 両方自動生成、ユーザーが選ぶ

### 方針
- 収益・フォロワー数は載せない(比較や自慢の道具にしない。コア哲学との一致)
- 出す情報の選択機能なし(MVPは固定)

### 生成タイミング
- ユーザーはいつでも自分で生成できる
- マイルストーン達成時に「シェアカード作りませんか?」と提案
- 毎月の節目にも「今月のシェアカード作りませんか?」と提案
- 提案は提案にとどめ、強制しない
