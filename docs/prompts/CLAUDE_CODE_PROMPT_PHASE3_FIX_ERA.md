# Claude Code 用プロンプト ─ フェーズ3 修正指示: 時代表示ルールの統一

このプロンプトは VSCode 内の Claude Code 拡張機能に渡す指示書です。
フェーズ3の実装で**街並みの時代表示ルールに不整合がある**ため、修正を依頼します。

---

## 現状の問題

現在の街並みは、**過去のすべての時代の建物が同時に並んで表示される**実装になっています。
例: 原始の小屋・古代の家・中世のレンガ建物・現代のビルが、同じ画面に同時に存在している。

これは設計意図と異なります。

---

## 正しい仕様(明確化)

### 大原則
**一つの画面では一つの時代の建物のみを表示する**。

ユーザーの現在の `current_era` に対応する建物**だけ**が街に建っているのが正しい状態です。
時代が進化した瞬間、街全体が新しい時代の見た目に**置き換わる**ように見えるべきです。

### 各時代の街の見え方

| 時代 | 街並みの構成 |
|---|---|
| primitive(原始) | 原始時代の建物だけが並ぶ |
| ancient(古代) | 古代時代の建物だけが並ぶ |
| medieval(中世) | 中世時代の建物だけが並ぶ |
| modern(近代) | 近代時代の建物だけが並ぶ |
| contemporary(現代) | 現代時代の建物だけが並ぶ |
| future(未来) | 未来時代の建物だけが並ぶ |

### 修正対象
- `lib/city/layout.ts`(街並み配置ロジック)
- `components/city/CityScene.tsx`(街並み全体の描画)
- `components/city/Building.tsx`(建物コンポーネント)
- 関連箇所すべて

建物の `type` は引き続き入力カテゴリ(X投稿・noteなど)と連動してOKですが、
`era` は**ユーザーの現在の current_era 一つのみ**にしてください。

---

## マイルストーン演出時の挙動

時代が進化する瞬間(Day 30/90/180/270/365 到達時):

1. 既存の街(旧時代の建物)が一旦フェードアウト or 暗転
2. フラッシュ演出(2秒、派手)
3. 新時代の建物が「ジャン!」と現れる
4. 街全体が新時代の見た目に置き換わる

> 当初のプロンプトでは「既存の建物は残る」と書いていましたが、これは**取り消し**です。
> 時代が変わったら街全体が置き換わるのが正しい設計です。

---

## 将来の振り返り機能について(伝達のみ、今フェーズでは実装しない)

**今後のフェーズで「振り返り画面」を実装する可能性があります**。

振り返り画面では「時代スライダー」「進捗スライダー」で過去の街並みを自由に見られる機能を予定しています。
つまり「過去の時代の街がどんな見た目だったか」を後から呼び出せる必要があります。

### このフェーズで意識してほしいこと
- 時代ごとの建物の見た目を **`lib/city/assets.ts` のような定数ファイルに完全に分離**しておく
- 「現在の時代の街を描く」関数は、引数として`era`を受け取り、その時代の街を描けるように作る
  - 例: `renderCityScene(era: Era, buildingCount: number, seed: string)` のような関数シグネチャ
- ユーザーの`current_era`を渡せば現在の街、別のeraを渡せば過去の街、と切り替えられる設計に

### このフェーズで **実装しない** こと
- 振り返り画面そのもの
- 過去のeraを呼び出すUI
- 時代を切り替えるスライダー

→ 今は**設計の準備だけ**してください(関数シグネチャと定数ファイルの分離)。

## 修正後の動作確認

修正完了後、以下を確認してください:

- [ ] 現在のユーザーの `current_era` がprimitiveなら、原始時代の建物のみが表示される
- [ ] デバッグツールで`current_era`を切り替えると、街全体の建物が置き換わる
- [ ] マイルストーン演出で時代が進化すると、街が一度暗転 → 新時代の街が現れる
- [ ] 建物の見た目を引数 `era` で切り替えられる関数構造になっている(将来の振り返り機能に備える)
