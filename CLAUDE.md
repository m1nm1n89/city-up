# Claude Code 共通ルール

このプロジェクトは個人事業主への挑戦の一環として開発されている発信系副業挑戦者向けの継続支援アプリです。

## ⚠️ フェーズ完了時の必須作業

各フェーズの実装が完了したら、完了報告を出す**前**に以下を必ず実行してください:

1. **`docs/prompts/README.md` の進捗表を更新する**
   - 該当フェーズの「状態」列を 🚧進行中 → ✅完了 に変更
2. 更新内容を含めて git にコミットする(任意)

これを実行しないと、ユーザーは現在の進捗を把握できなくなります。

## 開発の基本ルール

### セキュリティ
- セキュリティ運用ルールは `docs/knowledge/03a_Webアプリ_メイン_認証セキュリティ.md` を厳守
- 新しいテーブルを追加する際は必ず RLS を有効化する(Default Deny)
- `service_role` キーは絶対にクライアントコードや `NEXT_PUBLIC_*` 環境変数に混入させない
- コード提案前に Security Audit First の原則に従う

### 過去フェーズの参照
- 過去フェーズの実装方針は `docs/prompts/` 配下を参照
- 命名規則・ファイル構造・データモデルは過去フェーズを踏襲する
- 変更が必要な場合はユーザーに明示的に許可を取る

### 技術スタック
- Next.js + TypeScript + Tailwind CSS
- Supabase(Auth + Postgres + RLS)
- pnpm 11+(`minimumReleaseAge: 7200` 設定済み)
- Framer Motion(アニメーション)
- Cloudflare Workers + OpenNext(デプロイ先)

### パッケージ管理
- 新しいパッケージを追加する際は、5日以内に公開されたバージョンはインストールされない設定になっている
- インストール失敗時は、minimumReleaseAge制約に引っかかっている可能性がある
